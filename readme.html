What it does

This plugin provides an easy way for developers to control the logging of all PHP-related errors to a myDebug*.log file. After you've run the installation SQL file, there are two new values in your admin's Configuration->Logging:

  1. Report All Errors (Admin)?. Set this value to Yes to enable all PHP errors to be reported for admin-related pages.
  2. Report All Errors (Store)?. Set this value to Yes to enable all PHP errors to be reported for store-related pages.

Notes:

  1. Use caution when enabling this plugin on a live store as your store's performance will most likely be significantly reduced!
  2. The store-side debug-log files can be very large, so make sure that you clean them out periodically.

What Errors are Really Errors?

The first time you enable the reporting of all errors on your store, you might see a number of items similar to

PHP Notice:  Constant SOME_LANGUAGE_CONSTANT_NAME already defined in {DIR_FS_CATALOG}\includes\languages\english.php on line xxx

These "notice" messages are issued if your store has made a template-override change to the default language file (for this example, english.php). That's because Zen Cart first loads your template-override language file and then — because sometimes new messages are introduced with a new Zen Cart version — loads the default language file. These notices can be safely ignored.

Installation

There are core-file overwrites in this plugin; you should always backup your cart's database and files prior to making any changes.

  1. Unzip the plugin's package file and rename the YOUR_ADMIN folder to match your custom admin folder's name.
  2. Log into your Zen Cart admin, open the plugin's file /report_all_errors_install.sql, copy and paste the contents of that file into your admin's Tools->Install SQL Patches and press the Send button.
  3. Copy the files provided by the plugin to your Zen Cart:
    1. /includes/auto_loaders/config.report_all_errors.php
    2. /includes/auto_loaders/paypal_ipn.report_all_errors.php
    3. /includes/extra_configures/enable_error_logging.php1
    4. /includes/init_includes/init_report_all_errors.php
    5. /YOUR_ADMIN/includes/auto_loaders/config.report_all_errors_admin.php
    6. /YOUR_ADMIN/includes/extra_configures/enable_error_logging.php1
    7. /YOUR_ADMIN/includes/init_includes/init_report_all_errors_admin.php

1These files were core-file overwrites in v1.0.0 of this plugin and are now replaced by the similarly-named files from the Zen Cart v1.5.5 distribution to ensure that any previous version of this plugin is removed.

Uninstall

Replace the backup copies of the files you copied during the installation, delete the new files and run the plugin's file /report_all_errors_uninstall.sql.
